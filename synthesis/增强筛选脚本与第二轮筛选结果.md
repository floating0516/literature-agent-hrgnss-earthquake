# 增强筛选脚本与第二轮筛选结果

> 目的：记录对 `scripts/screen_candidates.py` 的增强，以及用增强版脚本重跑 broad candidate pool 后的结果。

---

## 1. 为什么要增强筛选脚本

上一轮扩大搜索得到 285 条候选后，原始筛选脚本存在一个问题：

> 规则主要偏向 GNSS / geodetic EEW，对 deep learning、machine learning、source characterization、finite-fault inversion 等方向不够敏感。

这会导致两类问题：

1. 真正有价值的 deep learning / source characterization 论文可能被低估；
2. seismic-only ML 论文和 GNSS/DL 交叉论文混在一起，后续不容易按主题选文献。

因此本轮选择方案 A：先增强筛选脚本，再重跑大规模候选池。

## 3. 重跑筛选结果

输入：

```text
papers/broad_candidates_combined.jsonl
```

输出：

```text
papers/broad_screened_candidates_v2.jsonl
papers/broad_selected_papers_v2.jsonl
papers/broad_screening_results_v2.md
papers/broad_screening_comparison_v2.md
papers/broad_priority_reading_list_v2.md
```

命令：

```bash
python3 scripts/screen_candidates.py \
  --input papers/broad_candidates_combined.jsonl \
  --output papers/broad_screened_candidates_v2.jsonl \
  --selected papers/broad_selected_papers_v2.jsonl \
  --report papers/broad_screening_results_v2.md \
  --comparison papers/broad_screening_comparison_v2.md
```

结果：

| Version | Keep | Maybe | Discard | Total |
|---|---:|---:|---:|---:|
| 原始 broad 筛选 | 82 | 100 | 103 | 285 |
| 增强 broad 筛选 v2 | 141 | 72 | 72 | 285 |

解释：

- Keep 数量增加，是因为 deep learning / source characterization 相关论文被更合理地提升；
- Discard 数量减少，说明原规则确实低估了一批非 GNSS 但方法相关的论文；
- 这不代表 141 篇都要精读，而是表示它们值得进入二级筛选或按主题池保留。

## 8. 当前仍存在的问题

### 8.1 Keep 数量偏多

增强后 Keep 从 82 增加到 141。作为候选池可以接受，但作为精读清单仍然太多。

解决方式：

- Keep 表示“值得保留”；
- 第二批精读仍应从 `broad_priority_reading_list_v2.md` 中选 5–10 篇；
- 后续可加入 LLM screening 或人工审核。

### 8.2 `deep_learning_eew` 和 `deep_learning_source_characterization` 有重叠

有些论文同时包含 deep learning、early warning、source characterization、rupture 等词，因此会被分到多个类别。

这不是错误，而是说明这类论文可能跨主题。后续可以进一步增加：

```text
primary_category
secondary_categories
```

### 8.3 `other` 类仍然偏多

`other` 中可能包含：

- 泛地震学论文；
- 高引用综述；
- 遥感/地表形变但非实时 GNSS；
- 方法参考但不是当前核心主题；
- OpenAlex abstract 缺失导致分类不足。

后续可以继续细化。

## 9. 架构结论

增强筛选后，当前文献 Agent 的候选池应按主题线维护，而不是只按一个总分排序：

```text
candidate_pool/
├── gnss_geodetic_eew
├── seismogeodesy_fusion
├── finite_fault_source_inversion
├── deep_learning_eew
├── deep_learning_source_characterization
├── tsunami_large_earthquake
└── other / manual_check
```

这样可以保证后续精读文献覆盖：

- 传统 GNSS baseline；
- seismogeodesy fusion；
- finite-fault/source inversion；
- deep learning 方法；
- 大震/海啸应用场景。
