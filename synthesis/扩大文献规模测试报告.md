# 扩大文献规模测试报告

> 目的：在三篇论文最小闭环已经跑通后，测试扩大文献搜索规模是否可行，并观察搜索速度、候选数量、筛选效果、主题覆盖、开放全文状态和下一批精读优先级。

---

## 1. 测试目标

本轮测试不直接下载大量 PDF，而是先扩大 metadata 搜索规模，验证以下问题：

1. OpenAlex 在更大 query 数量下速度是否仍可接受；
2. 当前粗筛规则在更广泛主题下是否还能过滤噪声；
3. 候选论文是否能覆盖 GNSS / seismogeodesy / deep learning 多条线；
4. Open-access PDF 状态是否足够支持下一轮 5–10 篇精读；
5. 下一批应优先阅读哪些论文。

---

## 2. 搜索设计

本轮将搜索分成两组，避免一次请求过多导致超时。

### 2.1 GNSS / geodetic / seismogeodesy 组

使用 8 个 query，每个 query 取 30 条：

```text
"high-rate GNSS" earthquake
"real-time GNSS" earthquake
"geodetic earthquake early warning"
"GNSS seismology" earthquake
"seismogeodetic" earthquake early warning
"G-FAST" GNSS earthquake
"REGARD" GNSS earthquake
"GNSS" "finite fault" earthquake
```

输出：

```text

papers/broad_search_summary_gnss.md
```

结果：

```text

耗时 58.31 秒
```

```text

### 2.3 合并结果

合并两个候选集并按 DOI / OpenAlex ID / title 去重：

```

---

## 3. 搜索速度结论

第一次尝试用 13 个 query、每个 50 条一次性搜索时，OpenAlex 请求在读取响应时发生 timeout。随后改为分组搜索：

| 搜索组 | Query 数 | 每个 query 数量 | 去重候选 | 耗时 |
|---|---:|---:|---:|---:|
| GNSS / geodetic | 8 | 30 | 182 | 58.31 s |
| Deep learning / ML | 5 | 25 | 114 | 44.44 s |
| 合并 | 13 | - | 285 | - |

结论：

> 扩大到 200–300 篇候选时，OpenAlex 仍然可用，但不建议一次性大请求。更稳妥的方式是按主题线分组搜索，每组控制 query 数和 per-page 数量。

- tsunami / 大震快速响应线能补充 Mw 8–9 俯冲带应用场景。

## 8. 当前粗筛规则的表现

### 8.1 优点

- 能快速处理 285 条候选；
- 能把明显相关的 GNSS / earthquake / finite-fault / EEW 论文推到 Keep；
- 能继续过滤 structural health monitoring、UAV、bridge monitoring 等噪声；
- 能保留 manual download 状态，避免因为 PDF 不可自动下载而丢掉重要论文。

### 8.2 问题

1. **Deep learning 线不够精细**  
   当前 `screen_candidates.py` 的强正向词主要为 GNSS / geodetic EEW，缺少 deep learning、machine learning、neural network、transformer、CNN、LSTM、source characterization 等专门词。

2. **Seismic-only ML 容易混入**  
   这类论文可能对方法有帮助，但不一定直接服务于 HR-GNSS，因此需要二级分类，而不是简单 keep/discard。

3. **高引用综述可能得分偏高**  
   由于规则中加入了 cited_by_count 加分，一些 broad review 或相邻主题论文会进入 maybe/keep，需要 LLM 精筛或人工判断。

4. **OpenAlex 的 PDF URL 仍需验证**  
   本轮只测试 metadata，不测试实际 PDF 下载成功率。

## 9. 对架构的影响

本轮测试说明，后续架构应该从“单一搜索结果列表”升级为“多主题线候选池”：

```text
candidate_pool/
├── gnss_geodetic_eew
├── seismogeodesy
├── finite_fault_source_inversion
├── deep_learning_eew
├── deep_learning_source_characterization
├── tsunami_large_earthquake
└── manual_download_needed
```

也就是说，不应该只按一个总 relevance score 排序，而应该按研究线组织：

1. GNSS baseline 线；
2. deep learning 方法线；
3. GNSS + seismic fusion 线；
4. 大震 / tsunami 应用线；
5. source inversion / finite fault 线。

这样后续综合时才能避免只读 GNSS baseline，而忽略 DL 方法论文。
