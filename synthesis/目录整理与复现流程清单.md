## 5. 当前可复现流程

### Step 0：确认主题配置

查看或修改：

```text
configs/topics.yaml
configs/sources.yaml
```

其中 `topics.yaml` 定义当前研究主题、关键词和筛选参数；`sources.yaml` 定义开放元数据源和合法全文策略。

---

### Step 1：搜索候选论文

运行：

```bash
python3 scripts/search_openalex.py
```

输出：

```text
papers/candidates.jsonl
papers/search_summary.md
```

当前已验证：

- 3 个 query；
- 每个 query 20 条；
- 去重后 44 篇候选；
- 总耗时约 7 秒。

---

### Step 2：粗筛候选论文

运行：

```bash
python3 scripts/screen_candidates.py
```

输出：

```text
papers/screened_candidates.jsonl
papers/selected_papers.jsonl
papers/screening_results.md
papers/screening_comparison.md
```

当前结果：

- Keep：21；
- Maybe：11；
- Discard：12。

该步骤主要过滤：

- structural health monitoring；
- UAV；
- bridge monitoring；
- cloud computing early warning；
- machine vision deflection monitoring。

---

### Step 3：确定核心论文并获取合法 PDF

根据：

```text
papers/core_papers_priority.md
papers/selected_papers.jsonl
```

选择优先阅读论文。

PDF 获取原则：

1. 优先开放获取 PDF；
2. 使用 OpenAlex / Unpaywall / arXiv / EarthArXiv / publisher OA；
3. 如果无法自动下载，标记 manual download；
4. 用户可通过学校/机构权限合法下载后放入：

```text
papers/raw_pdf/
```

注意：

> 本流程不绕过 paywall，不使用 API key 规避出版社权限。

---

### Step 4：解析 PDF

运行：

```bash
python3 scripts/parse_pdfs.py
```

输入：

```text
papers/raw_pdf/*.pdf
```

输出：

```text
papers/parsed_md/*.md
papers/pdf_parse_log.md
```

当前解析工具：

```text
pdftotext -layout -enc UTF-8
```

适合 MVP，但双栏顺序、公式、图表可能不完美。

## 6. 当前流程一句话版

```text
先用 OpenAlex 快速搜 metadata，再用规则粗筛去噪；对核心论文通过开放获取或人工合法下载 PDF；用 pdftotext 转成 Markdown；用精读 prompt 生成 reading notes；用综合 prompt 做跨文献 synthesis；最后把 reading notes 和 synthesis 切成 rag/chunks.jsonl，作为后续检索和写作的知识库基础。
```

## 7. 后续建议

下一步如果继续扩展，建议按这个顺序：

1. 增加 deep learning earthquake magnitude/source characterization 搜索线；
2. 第二批下载并精读 5–10 篇；
3. 为 `rag/chunks.jsonl` 增加关键词检索脚本；
4. 再接 embedding + Chroma/LanceDB；
5. 加入 citation verification：每个关键数值回查 PDF 页码、图号、表号；
6. 基于 RAG 生成正式综述初稿。
