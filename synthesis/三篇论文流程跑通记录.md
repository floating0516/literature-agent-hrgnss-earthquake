# 三篇论文流程跑通记录

> 目的：记录文献阅读 Agent 原型从搜索、筛选、下载、解析到单篇精读卡片的第一轮闭环结果。

---

## 1. 本轮目标

本轮目标不是追求文献规模，而是用 3 篇论文验证完整流程是否可运行：

```text
OpenAlex 搜索 → 粗筛 → 核心论文优先级 → PDF 下载 → PDF 解析 → 阅读卡片
```

本轮选择的 3 篇论文均围绕 real-time GNSS / HR-GNSS / geodetic EEW / rapid source characterization。

## 4. 关键观察

### 4.1 搜索阶段很快

OpenAlex 搜索 3 个 query、每个 20 条，去重得到 44 篇候选，总耗时约 7 秒。

说明：

> 第一版 metadata 搜索不是瓶颈，可以继续使用 OpenAlex 作为轻量入口。

### 4.2 粗筛能有效去噪

原始候选中包含结构健康监测、UAV、桥梁监测、云计算 early warning 等噪声。粗筛后：

- Keep：21；
- Maybe：11；
- Discard：12。

噪声过滤记录见：

```text
papers/screening_comparison.md
```

### 4.3 OpenAlex PDF URL 有用，但不总能脚本下载

成功下载 3 篇 PDF，但 MDPI Maduo HR-GNSS 论文自动下载失败，原因是 HTTP 403 或返回 HTML。

说明：

> 自动下载失败不代表论文不可用，应标记为人工下载或后续换方式下载。

### 4.4 pdftotext 可以快速跑通解析

当前使用：

```text
pdftotext -layout -enc UTF-8
```

优点：

- 本地已有工具；
- 速度快；
- 能提取大部分正文。

局限：

- 双栏顺序可能局部混乱；
- 图表、页眉页脚、参考文献会混入正文；
- 公式解析不理想；
- 后续需要更高质量时可引入 GROBID、Marker 或 PyMuPDF。

### 4.5 精读卡片结构可用

三篇阅读卡片均按统一字段输出：

- Metadata；
- One-sentence summary；
- Research problem；
- Data；
- Method；
- Evaluation metrics；
- Key results；
- Strengths；
- Limitations；
- Relation to my research；
- Useful citations；
- Open questions；
- Tags；
- TODO verify。

说明：

> 当前 prompt 模板可以支持结构化知识库积累。

## 5. 三篇论文对当前研究的互补价值

| 论文 | 主要价值 | 对 HR-GNSS + DL 研究的启发 |
|---|---|---|
| Crowell et al. 2016 G-FAST | 系统框架和 geodetic EEW baseline | PGD/CMT/finite-fault 可作为传统 baseline；latency/noise/dropout 是关键鲁棒性指标 |
| Kawamoto et al. 2016 REGARD | 实时 GNSS finite-fault operational case | 可借鉴其 Mw、VR、fault geometry、近场台站依赖等评价方式 |
| Melgar et al. 2019 Ridgecrest | real-time HR-GNSS 数据质量验证 | 可用于设计 HR-GNSS DL 输入特征、噪声增强、latency 和 PGD/offset 不确定性 |

## 6. 当前问题

1. 文献数量太少，只能证明流程可行，不能支撑系统综述；
2. 第一批搜索主要偏 geodetic EEW 和实时 GNSS，深度学习文献还未覆盖；
3. PDF 解析质量可以接受但不够精细；
4. 阅读卡片目前仍需要人工核查数值和页码；
5. 还没有构建 RAG chunk 和向量库；
6. 还没有做三篇之间的跨文献综合。
